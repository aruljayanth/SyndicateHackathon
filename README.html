Guidance for App

  1. Install the Apk file
  2. click the book visit button in the interface
  3. The alarm button(float button) is used to create an alarm for the visit time
  4. Once you click the book visit you will get the book your visit intent
  5. Type the name phone number and email id
  6. Select your required service (only a few listed)
  7. Select the date and time and then clicck the book button (validation for the fields is not done it would be done before the onsite hackathon)
  8. Once you click the book button a qr code will be generated (scan the qr code when you reach the bank)

Guidance for web API

  1. index.php:
    Once the user registers in the app, the details are pushed to database (table name: details) and the token number is sent as response to the app to generate qr code.
  2. custbank.php:
    The qr code will be scanned in the bank using raspberry pi. The scanned token number will be sent to custbank.php in json format , and pushed to database with the details about the counter number, token number and time_in will be put inside the database (table name: current).

Guidance for Hardware - Demonstration is in demo video

  1. Scan the generated QR code from the app
  2. The Token number in the code is sent to a web API and thus updated in the database, stating that the user has entered the bank
  3. QR code cannot be scanned before the time selected by the user while registration

Guidance for Bankers Webpage

  1. Open current.php
  2. Select the counter and click submit
  3. Page will be redirected to history.php with some query strings
  4. The current user is displayed in the webpage
  5. If the end button is clicked, The user exits the counter and next person in the counter is automatically displayed
  6. In Backend: The person is deleted from Table name:Current and pushed to Table:History, for bank's reference and to predict the time for that counter
  7. Timeout is also pushed to the Table: History
